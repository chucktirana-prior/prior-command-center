Location Monitor — Full Flow

How guide post locations and article links both flow through the same check pipeline

Import from Contentful
Runs both imports in parallel via Promise.all()
Source A — Articles
importLocationBusinessesFromContentful()
Scans articleBody markdown for external links → filters out non-venue patterns (book now, enquiry…) → extracts business_name website_url section
Source B — Guide Posts ✨ New
importGuidePostsFromContentful()
Fetches all guidePost entries → batch-resolves parent guide title & slug → batch-resolves guideType for section label → reads title website address phoneNumber
Article result row
article_id · article_title · section · business_name · website_url · source_type = "article"
Guide post result row
article_id = guide's Contentful ID · article_title = guide title · section = guideType name · address & phone pre-populated · source_type = "guide_post"

If no website → fallback URL: priorworld.com/guides/{guide}/{post}
SQLite — location_places
upsertLocationPlace()
UNIQUE(article_id, website_url) source_type address seed phone seed
On conflict: updates metadata, preserves any existing Maps/website/AI check results
Check pipeline — runs on all rows regardless of source
🗺️
Google Maps Check
Searches by maps_query (name + address). Guide posts get more precise results because address is pre-seeded from Contentful.
🌐
Website Check
Fetches website_url, scans for closure signals. Guide posts with a real venue website are fully checked. Prior-URL fallbacks return "active".
🤖
AI Review
Claude reads website evidence + Maps status and gives a confidence verdict. Works identically for both source types.
